Introduction¶

In this notebook, the goal is attempting to use ARMA models to predict the share of a company. ARMA models are used for time-series analysis, but may not be optimal to use for this case. However, this is more or less a demonstration of how such concepts can be applied in a real-life scenario. The shares prices for the company AMD (Advanced Micro Devices), from 14:20 to 15:48 on October 30, 2020. The data will be provided from Alpaca's API for market data. The data is by per-minute bars.

Initialization¶

Start by importing the necessary libraries. In this case, the libraries that will be used are NumPy, Pandas, MatPlotLib StatsModels and Requests. Although Alpaca has it's own package for Python functionality, It would be more fair to be platform-neutral and do it the "old-fashioned" way. This part will cover the whole importing and processing data procedures. It will cover the library importing process, data importing process, data handling and processing procedures, and finally preparing the data for analysis.

Starting Analysis¶

View the data and explore any patterns or conditions. This can be done with matplotlib.

In [5]:
amd = plt.plot(data['datetime'], data['price'])

What's clear in this dataset is that it violates almost all conditions required to run an ARMA model, so what can be done is try to convert it into a stationary process using several transformations. To start, try to normalize the dataset by converting the values into standardized z scores. This is simple but should be very effective.

In [6]:
mu = np.mean(data['price'])
sigma = np.std(data['price'])

data['data_norm'] = (data['price'] - mu) / sigma
plt.plot( data['datetime'], data['data_norm'])
plt.axhline(0, color = 'black', linestyle = '--')
Out[6]:

According to the plot, almost all of the values are more or less between 1.5 standard deviations from the mean, which is good since it indicates that there is low variance. However, the data has still remained the same. Another step towards transformation would be differencing, as explained in here in this document provided by Texas A&M.

The data now seems to have a much better pattern than before. Check for stationarity using The Augmented Dickey-Fuller Unit-Root Test. Looking at the statsmodels ADF documentation, the $1^{st}$ value of the array is the ADF value, and the $2^{nd}$ value is the p-value. If the p-value is below 0.05, it means that value is significant enough to assume that the data is stationary. The returning output is not the most user-friendly, so creating a function to better organize the output for relevant information here is a good idea.

In [8]:
def adf_test(arr):
    
    res = sm.tsa.stattools.adfuller(arr.dropna())
    print('ADF value is: ', res[0])
    print('p-value is: ', float(res[1]))

adf_test(data['diff'])

Given the PACF and ACF, it is apparent that for the most accurate results, the model should only predict up to 1 lag at most. The next step will be building the model, fitting it, plotting it, and compare it to the actual results.

Building An ARMA Model¶

The model will be built using the statsmodels ARIMA features. This is straight-forward and can be done by:

In [10]:
model = sm.tsa.arima.ARIMA(data['diff'], order = (1, 0, 1))

Basically, this generates an ARMA model because the second order value is the difference, which is 0 in this case because the model should be an ARMA model. The 1's are the order of MA and AR. The MA and AR were chosen to be of the $1^{st}$ order for the best results. Now, fit the model and forecast the results by:

In [11]:
model_fit = model.fit()
model_fit.forecast(1)
Out[11]:
89   -0.043091
dtype: float64

The next predicted value should be -0.04091. However, this does not help at all because this is the predicted value of the normalized difference between consecutive values. To obtain the predicted price, it is needed to reverse the transformation process. Because this was not a lengthy transformation, what can be done is simply add the predicted value to the lagging normalized score, then multiply that value by sigma and then add the mean. Essentially inverting the transformation process. The whole formula should be:

$\hat{Y}_{n} = \sigma*(\hat{v_{n}} + norm_{n-1}) + \mu $

Where $\hat{Y}_{n}$ is the predicted price, $\hat{v_{n}}$ is the forecasted normalized difference (the value the model forecasts), and $norm_{n-1}$ is the normalized value of the previous (lagging) data point. Writing a function to obtain the price given this equation would be a good step to take here.

In [12]:
def price_pred(val):
    res = sigma * (val + data['data_norm'][len(data)-1]) + mu
    return res

price_pred(-0.043091)
Out[12]:
74.78564597534005

This is the final output. It is the price that the fitted model forecasts given the data. This function could also be easily scaled if it should predicted more lags.
